# Excelに「設定を反映する」ボタンを追加する手順

この手順では、元の `excel/sample-company.xlsx` を直接変更しません。マクロ有効版を別ファイルとして作成し、そこにVBAモジュールと操作メニューを追加します。

## 1. `.xlsx` と `.xlsm` の違い

| 形式 | 用途 |
| --- | --- |
| `.xlsx` | 通常のExcelファイルです。マクロは保存できません。元テンプレートとして扱います。 |
| `.xlsm` | マクロ有効ブックです。Excel内のボタンから処理を実行したい場合に使います。 |

元ファイルを守るため、`excel/sample-company.xlsx` はそのまま残し、マクロ版は `excel/sample-company.xlsm` として保存してください。

## 2. マクロ有効版を作成する

1. `excel/sample-company.xlsx` をExcelで開きます。
2. 「ファイル」→「名前を付けて保存」を選びます。
3. 保存先は `excel/` フォルダのままにします。
4. ファイル名を `sample-company.xlsm` にします。
5. ファイルの種類で「Excel マクロ有効ブック (*.xlsm)」を選びます。
6. 保存します。

想定配置は次のとおりです。

```text
プロジェクトルート/
├── run-all.bat
└── excel/
    ├── sample-company.xlsx
    └── sample-company.xlsm
```

## 3. VBAモジュールをインポートする

`scripts/vba/ConcurBotOperations.bas` は、Windows版ExcelのVBEへそのままインポートできるように、日本語メッセージを `ChrW()` で組み立てる形式にしています。UTF-8の日本語リテラルには依存していません。

1. `excel/sample-company.xlsm` をExcelで開きます。
2. `Alt + F11` を押してVBAエディターを開きます。
3. 左側のプロジェクト一覧で `sample-company.xlsm` を選びます。
4. 「ファイル」→「ファイルのインポート」を選びます。
5. `scripts/vba/ConcurBotOperations.bas` を選択してインポートします。
6. VBAエディターを閉じます。
7. Excelを保存します。

すでに古い `ConcurBotOperations.bas` を取り込んでいてMsgBoxが文字化けしている場合は、VBAエディター上で古いモジュールを削除してから、最新の `scripts/vba/ConcurBotOperations.bas` を再インポートしてください。

## 4. `00_操作メニュー` シートを作成する

1. Excelで新しいシートを追加します。
2. シート名を `00_操作メニュー` に変更します。
3. 次の内容を入力します。

| セル | 内容 |
| --- | --- |
| A1 | Concur迷子防止Bot 操作メニュー |
| A3 | 設定編集後に以下のボタンを押してください |
| A5 | 設定を反映する |
| A7 | Excel更新 → config生成 → HTMLレポート生成 → Bot起動 |

必要に応じて、A1を大きめの太字にし、A5付近にボタンを配置してください。

## 5. ボタンを配置してマクロを割り当てる

1. Excelの「開発」タブを表示します。
2. 「挿入」→「フォーム コントロール」→「ボタン」を選びます。
3. `00_操作メニュー` シート上にボタンを配置します。
4. マクロ一覧から `RunAllProcesses` を選びます。
5. ボタンの表示名を `設定を反映する` に変更します。
6. Excelを保存します。

「開発」タブが表示されていない場合は、「ファイル」→「オプション」→「リボンのユーザー設定」で「開発」にチェックを入れてください。

## 6. 実行時の動き

`設定を反映する` ボタンを押すと、VBAマクロ `RunAllProcesses` が次の処理を行います。

1. 現在のExcelファイルを保存します。
2. Excelファイルの保存場所からプロジェクトルートを特定します。
3. プロジェクトルート直下の `run-all.bat` を探します。
4. `run-all.bat` があれば起動します。
5. Excel更新、config生成、HTMLレポート生成、React画面起動が順番に実行されます。

## 7. 「コンテンツの有効化」について

マクロ有効ブックを開くと、Excel上部にセキュリティ警告が表示されることがあります。

このプロジェクト内で作成した `sample-company.xlsm` を利用する場合は、「コンテンツの有効化」を押すとボタンを実行できます。

会社PCのポリシーでマクロが禁止されている場合は、無理に有効化せず、プロジェクト直下の `run-all.bat` をダブルクリックして実行してください。

## 8. `run-all.bat` が見つからない場合

次の点を確認してください。

- `sample-company.xlsm` が `excel/` フォルダ内に保存されていること
- `run-all.bat` がプロジェクトルート直下にあること
- フォルダ構成が次の形になっていること

```text
プロジェクトルート/
├── run-all.bat
└── excel/
    └── sample-company.xlsm
```

Excelファイルを別フォルダへコピーして使う場合は、`run-all.bat` や `scripts/` などプロジェクト一式も同じ構成で配置してください。

## 9. 補足

- この手順では `excel/sample-company.xlsx` は変更しません。
- `.xlsm` はマクロ実行用の別ファイルとして扱います。
- Codex環境ではExcelのGUI操作やマクロ実行確認はできないため、実際のボタン配置とクリック確認はWindows版Excelで行ってください。
